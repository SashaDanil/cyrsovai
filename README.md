# Система управления службой доставки

Простое веб-приложение для управления службой доставки, разработанное на Flask с использованием SQLite базы данных.

## 📋 Описание проекта

Веб-приложение позволяет управлять основными сущностями службы доставки:
- Курьеры
- Клиенты  
- Товары
- Заказы

Приложение предоставляет возможности просмотра и добавления записей в базу данных через веб-интерфейс.

## 🛠 Технологии

- **Backend**: Python 3, Flask
- **Database**: SQLite
- **Frontend**: HTML (без CSS на данный момент)

## 📁 Структура проекта

```

## 🚀 Установка и запуск

### 1. Клонирование репозитория

```bash
git clone <url-репозитория>
cd delivery-service
```

### 2. Установка зависимостей

Установите необходимые пакеты Python:

```bash
pip install flask
```

Или, если у вас есть файл `requirements.txt`:

```bash
pip install -r requirements.txt
```

### 3. Создание базы данных

Запустите скрипт для создания и заполнения базы данных тестовыми данными:

```bash
python create_db.py
```

Это создаст файл `delivery_service.db` с таблицами и начальными данными.

### 4. Запуск приложения

```bash
python app.py
```

Приложение будет доступно по адресу: `http://127.0.0.1:5000`

## 🌐 Функциональность

### Основные возможности

1. **Управление курьерами**
   - Просмотр списка всех курьеров
   - Добавление новых курьеров с указанием:
     - Личных данных (имя, фамилия, телефон, email)
     - Статуса занятости
     - Транспортного средства
     - Рейтинга

2. **Управление клиентами**
   - Просмотр списка клиентов
   - Добавление новых клиентов с указанием:
     - Личных данных
     - Адреса доставки
     - Бонусных баллов
     - Категории (обычный/постоянный/VIP)

3. **Управление товарами**
   - Просмотр каталога товаров
   - Добавление новых товаров с указанием:
     - Названия и описания
     - Цены и веса
     - Категории
     - Количества на складе
     - Рейтинга

4. **Управление заказами**
   - Просмотр всех заказов с информацией о клиенте и курьере
   - Создание новых заказов с выбором:
     - Клиента
     - Курьера (опционально)
     - Суммы заказа
     - Способа оплаты
     - Статуса

## 📝 Примечания

- База данных создается автоматически при первом запуске `create_db.py`
- Приложение использует SQLite, что не требует настройки отдельного сервера базы данных
- Все данные сохраняются в файл `delivery_service.db`
- Для тестирования приложения созданы начальные данные (10 курьеров, 10 клиентов, 10 товаров, несколько заказов)
